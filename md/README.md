# md

This directory is the working-memory area for the repository.

Use it for:
- current status and run notes
- implementation plans
- architecture notes
- discussions and decisions worth preserving in-repo

Preferred conventions:
- `running.md` tracks current state, common commands, and known gaps
- `plan.md` tracks the active roadmap
- add focused docs when a topic deserves its own file
- move superseded plans to `md/old/` if they are still useful for reference

Do not let important project context live only in chat history when it should be preserved here.
